# bwai-jogja-online
# Building ADK Agents with Antigravity and Agent Skills

This repository demonstrates how to build and test a custom **Travel Assistant** AI Agent using the Google Agent Development Kit (ADK) and the `adk web` development UI, built seamlessly with Antigravity agent skills.

## Features Added
- **Mock Custom Tools**: Integrated Python functions (`search_google_maps_places`, `get_google_maps_directions`) that the LLM invokes to fetch localized attraction data and structural transit routes.
- **Agent Initialization**: Implemented an `LlmAgent` using `gemini-2.5-flash` with a strict system prompt tailored for travel itinerary generation.
- **ADK Web UI Server Support**: Restructured the project into the ADK-compliant `agents/` directory pattern so the agent scales readily to an interactive web frontend.

---

## Directory Structure

To correctly utilize `adk web` loading conventions, the project is structured to export a variable named `root_agent`:

```text
├── agents/ 
│   └── travel_assistant/
│       ├── __init__.py     # Exports the root_agent: `from .agent import root_agent`
│       ├── agent.py        # Contains the core tools and LlmAgent definition logic
│       └── .env            # Environment variables (e.g., GOOGLE_API_KEY)
└── README.md
```

## Workshop Workflow: Utilizing Agent Skills

As part of the workshop, this agent was entirely conceptualized and generated by equipping the Antigravity AI assistant with a dedicated **ADK Agent Skill**. By combining AI capabilities with domain-specific knowledge, rapid prototyping becomes effortless.

### Creating the Agent Skill

To build the foundation, we first instructed Antigravity to create a reusable agent skill that automatically stays up to date with ADK's core documentation. You can replicate it by running this prompt:

> "Create a new, reusable agent skill for the Google Agent Development Kit (ADK). Implement this skill within the `.agents/skills/adk-skill` directory, adhering to the structural guidelines provided by the Antigravity documentation and the best practices outlined in the ADK's `llms-full.txt`. Ensure the skill is portable by using only relative paths and implement functionality to automatically update the specified reference file upon each execution."

Once created, you simply select this skill from your UI before asking the AI to build your ADK apps!

### Creating the Agent

Instead of manually coding step-by-step, you can simply equip the AI with the ADK Agent Skill and provide the following prompt:

> "Create a travel assistant ADK agent that builds a personalised itinerary for a given location based on the user’s interests, available time, and budget. Based on their interests, use Google Maps to find attractions and restaurants, organise them into a day-by-day plan with optimised routes and transport directions, and output the complete itinerary at once without asking for confirmation"

The AI will parse the latest ADK documentation, write the required code, structure your folders, compile mock tools, and squash runtime errors automatically!

## Setup Instructions

Follow these steps to set up the local environment and interact with the agent via ADK Web:

### 1. Create a Python Virtual Environment
We recommend utilizing a virtual environment to keep dependencies clean:
```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 2. Install Required Dependencies
You will need the `google-adk` framework. Additionally, running the `adk web` server requires `python-multipart` to parse form-data input correctly.
```bash
pip install google-adk python-multipart
```

### 3. Add Your Target API Key
Because the `travel_assistant` agent depends on Gemini models, it requires a valid API key. Create a `.env` file within the `agents/travel_assistant/` directory. 

```bash
# In agents/travel_assistant/.env
GOOGLE_API_KEY="PASTE_YOUR_API_KEY_HERE"
```

*Note: ADK automatically detects the `.env` file within the agent's folder and dynamically loads it into the environment!*

### 4. Run the ADK Web Server
With the environment built and the agent structured correctly, host the server to interact with the LLM via an advanced UI. Launch `adk web` from the workspace root by pointing to the `agents/` directory:

```bash
adk web --port=8080 agents/
```

Navigate to [http://127.0.0.1:8080](http://127.0.0.1:8080) in your web browser. You can now chat directly with the agent, test prompts, trace its function calls to the Google Maps mock tools, and view response latency!
